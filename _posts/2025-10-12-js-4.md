## 여러 값을 한꺼번에 저장하는 방법

지금까지 우리는 변수 하나에 값 하나를 저장했습니다:

```javascript
const name = "홍길동";
const age = 25;
const city = "서울";
```

하지만 관련된 데이터가 많아지면 어떻게 할까요?

```javascript
// 학생 10명의 이름을 저장한다면?
const student1 = "홍길동";
const student2 = "김철수";
const student3 = "이영희";
// ... 
```

이럴 때 사용하는 것이 '배열(Array)'과 '객체(Object)'입니다.

| 구분 | 배열 | 객체 |
|------|------|------|
| **저장 방식** | 순서대로 저장 | 이름(키)과 함께 저장 |
| **접근 방법** | 숫자 인덱스 [0], [1], [2]... | 속성 이름 .name, .age... |
| **사용 상황** | 같은 종류의 데이터 여러 개 | 관련된 다양한 데이터 묶음 |

## 배열 (Array) 기초

### 배열이란?

하나의 변수에 **여러 개의 값을 순서대로** 저장하고 싶을 때 사용합니다.

```javascript
// 배열 없이 (비효율적)
const student1 = "홍길동";
const student2 = "김철수";
const student3 = "이영희";
const student4 = "박민수";

// 배열 사용 (효율적)
const students = ["홍길동", "김철수", "이영희", "박민수"];
```

### 배열 만들기

```javascript
// 방법 1: 대괄호 사용 (일반적)
const fruits = ["사과", "바나나", "오렌지"];

// 방법 2: Array 생성자
const numbers = new Array(1, 2, 3, 4, 5);

// 빈 배열
const empty = [];
```

### 배열 요소 접근하기

배열의 각 요소는 인덱스(index)로 접근합니다. **인덱스는 0부터 시작합니다!**

배열의 요소 개수는 length 속성에서 확인할 수 있는데, 인덱스가 0부터 시작하기 때문에 가장 마지막 요소의 인덱스는 length-1 이 됩니다. 

```javascript
const fruits = ["사과", "바나나", "오렌지", "포도"];

console.log(fruits[0]);  // 사과
console.log(fruits[1]);  // 바나나
console.log(fruits[2]);  // 오렌지
console.log(fruits[3]);  // 포도

// 배열의 길이
console.log(fruits.length);  // 4

// 마지막 요소
console.log(fruits[fruits.length - 1]);  // 포도
```

### 배열 요소 수정하기

```javascript
const fruits = ["사과", "바나나", "오렌지"];

fruits[1] = "딸기";
console.log(fruits);  // ["사과", "딸기", "오렌지"]
```

## 객체 (Object) 기초

### 객체란?

**관련된 데이터를 이름(키)와 함께 저장하는** 자료구조입니다.

배열은 순서(인덱스)로 값을 찾지만, 객체는 **이름(키)**으로 값을 찾습니다. 이것이 가장 큰 차이점입니다.

**배열의 문제점**

```javascript
// 배열로 사용자 정보 저장 (헷갈림)
const user1 = ["홍길동", 25, "서울"];

// 이게 이름? 나이? 주소?
console.log(user[0]);  // "홍길동"
console.log(user[1]);  // 25
console.log(user[2]);  // "서울"
```<img width="774" height="227" alt="arr-1" src="https://github.com/user-attachments/assets/93b6921f-6432-404b-b936-b623785257ba" />


배열만 봐서는 각 값이 무엇을 의미하는지 알기 어렵습니다. 순서를 기억해야 하고, 중간에 데이터가 추가되면 인덱스가 모두 바뀝니다.

**객체로 해결**

```javascript
// 객체로 사용자 정보 저장 ( 명확함)
const user2 = {
  name: "홍길동",
  age: 25,
  city: "서울"
};

// 의미가 명확함!
console.log(user.name);  // "홍길동"
console.log(user.age);   // 25
console.log(user.city);  // "서울"
```

이제 각 값이 무엇을 의미하는지 명확하게 알 수 있습니다. 순서도 중요하지 않습니다.

### 객체 만들기

객체를 만드는 방법은 여러 가지가 있습니다.

#### 방법 1: 객체 리터럴 (가장 일반적)

객체 리터럴(Object Literal)은 중괄호 `{}`를 사용하여 객체를 만드는 방법입니다. 가장 간단하고 많이 사용하는 방법입니다.

**기본형**

```
const 객체이름 = {
  키1: 값1,        // 속성 (property)
  키2: 값2,        // 쉼표로 구분
  키3: 값3         // 마지막은 쉼표 생략 가능
};
```
- **키(key)**: 속성의 이름 (name, age, city 등)
- **값(value)**: 속성에 저장된 데이터
- **속성(property)**: 키와 값의 쌍

객체를 만들 때 가장 먼저 고려해야 할 방법입니다. 코드가 간결하고 읽기 쉬우며 여러 줄로 작성하면 가독성이 더 좋습니다

```javascript
// 한 줄로 작성 (데이터가 적을 때)
const point = { x: 10, y: 20 };

// 여러 줄로 작성 (데이터가 많을 때, 권장)
const student = {
  name: "홍길동",
  age: 20,
  grade: "A",
  major: "컴퓨터공학"
};
```

#### 방법 2: 빈 객체 만들기

빈 객체를 만들고 나중에 속성을 추가하는 방법입니다. 이 때 빈 중괄호 `{ }`를 사용해서 빈 객체를 만듭니다. 

```javascript
// 빈 객체 생성
const user = {};

// 나중에 속성 추가
user.name = "이영희";
user.age = 28;
user.email = "lee@example.com";

console.log(user);
// { name: "이영희", age: 28, email: "lee@example.com" }
```

이 방법은 처음에 어떤 속성이 필요한지 모를 때 유용합니다. 그리고 조건에 따라 다른 속성을 추가해야 할 때도 사용합니다

```javascript
const settings = {};

// 조건에 따라 속성 추가
if (userWantsSound) {
  settings.sound = true;
}

if (userWantsNotification) {
  settings.notification = true;
}
```

#### 방법 3: new Object() 생성자

빈 객체를 만들 때 `new Object()` 생성자를 사용할 수도 있습니다. 역시 빈 객체를 만든 후 속성을 하나씩 추가하는 방법입니다.

```javascript
const person = new Object();
person.name = "김철수";
person.age = 30;
person.city = "부산";

console.log(person);
// { name: "김철수", age: 30, city: "부산" }
```

new Object() 생성자를 사용한 방법은 객체 리터럴보다 코드가 길고 복잡합니다. 특별한 이유가 없다면 이 방법은 잘 사용하지 않죠.

주로 동적으로 객체를 만들어야 할 경우 이 방법을 사용할 수 있습니다. 하지만 이 경우에도 빈 객체 리터럴 `{ }`을 사용하는 것이 더 일반적입니다.

### 객체의 속성 이해하기

객체에서 키(key)와 값(value) 한 쌍을 속성(property)이라고 합니다. 

```javascript
const car = {
  brand: "현대",      // 속성 1: brand는 키, "현대"는 값
  model: "소나타",    // 속성 2: model는 키, "소나타"는 값
  year: 2023,        // 속성 3: year는 키, 2023은 값
  color: "흰색"      // 속성 4: color는 키, "흰색"은 값
};
```

**키(key) 작성 규칙:** 

키 이름은 일반적으로 따옴표 없이 작성하는데 공백이나 특수문자가 있으면 따옴표로 감싸야 합니다. 그리고 숫자로 시작할 경우에도 따옴표로 감싸야 합니다.

가능하면 간단한 이름을 사용하고 따옴표는 피하는 것이 좋습니다.

```javascript
const obj = {
  // 일반적인 키
  name: "홍길동",
  age: 25,
  
  // 공백이 있는 키 (따옴표 필요)
  "full name": "홍길동",
  
  // 특수문자가 있는 키 (따옴표 필요)
  "email-address": "hong@example.com",
  
  // 숫자로 시작하는 키 (따옴표 필요)
  "1st-place": true
};
```

**값(value)의 타입:**
- 객체의 값은 **모든 자료형**이 가능합니다

```javascript
const data = {
  // 문자열
  name: "홍길동",
  
  // 숫자
  age: 25,
  
  // 불린
  isStudent: true,
  
  // 배열
  hobbies: ["독서", "영화", "운동"],
  
  // 객체 (중첩 가능!)
  address: {
    city: "서울",
    district: "강남구"
  },
  
  // null
  middleName: null,
  
  // undefined
  nickname: undefined
};
```

### 객체 속성 접근하기

객체의 속성에 접근하는 방법은 두 가지입니다.

#### 방법 1: 점 표기법 (Dot Notation)

점(`.`)을 사용하여 속성에 접근하는 방법입니다. **가장 일반적이고 읽기 쉬운 방법**입니다.

코드가 깔끔하고 읽기 쉽기 때문에 대부분의 경우 이 방법을 사용합니다

```javascript
const person = {
  name: "홍길동",
  age: 25,
  city: "서울"
};

console.log(person.name);  // "홍길동"
console.log(person.age);   // 25
console.log(person.city);  // "서울"
```

점 표기법의 경우 속성의 키를 직접 써야 하는데 공백이나 특수문자가 있는 속성 키는 접근할 수 없습니다. 

```javascript
const obj = {
  "full name": "홍길동"
};

// 에러 발생
console.log(obj.full name);  // 문법 오류!
```

#### 방법 2: 대괄호 표기법 (Bracket Notation)

대괄호(`[]`)와 문자열을 사용하여 속성에 접근하는 방법입니다. 변수를 사용하여 동적으로 속성에 접근할 수 있고, 공백이나 특수문자가 있는 속성명도 접근할 수 있습니다. 

```javascript
const person = {
  name: "홍길동",
  age: 25,
  city: "서울"
};

console.log(person["name"]);  // "홍길동"
console.log(person["age"]);   // 25
console.log(person["city"]);  // "서울"
```

**언제 대괄호 표기법을 사용할까?**

**1. 변수로 속성에 접근할 때** 어떤 속성을 가져와야 하는지 변수에 지정해 놓고 변수를 사용해 접근할 수 있습니다. 

```javascript
const person = {
  name: "홍길동",
  age: 25,
  city: "서울"
};

// 어떤 속성을 가져올지 변수로 결정
const key = "name";
console.log(person[key]);  // "홍길동"

// 다른 속성으로 변경
const key2 = "age";
console.log(person[key2]);  // 25

// 점 표기법에서는 불가능
console.log(person.key);  // undefined (person에 key라는 속성을 찾음)
```

**2. 속성 이름에 공백이나 특수문자가 있을 때** 

```javascript
const user = {
  "full name": "홍길동",
  "email-address": "hong@example.com",
  "1st-place": true
};

console.log(user["full name"]);       // "홍길동"
console.log(user["email-address"]);   // "hong@example.com"
console.log(user["1st-place"]);       // true
```

**3. 반복문에서 모든 속성에 접근할 때**

```javascript
const person = {
  name: "홍길동",
  age: 25,
  city: "서울"
};

// for...in 문으로 모든 속성 순회
for (let key in person) {
  console.log(\`\${key}: \${person[key]}\`);
}
// 출력:
// name: 홍길동
// age: 25
// city: 서울
```

#### 점 표기법 vs 대괄호 표기법 비교

| 상황 | 점 표기법 | 대괄호 표기법 |
|------|----------|-------------|
| 일반적인 경우 | `person.name` | `person["name"]` |
| 변수 사용 | 불가능 | `person[key]` |
| 공백이 있는 속성 | 불가능 | `person["full name"]` |
| 특수문자 있는 속성 | 불가능 | `person["email-address"]` |
| 숫자로 시작하는 속성 | 불가능 | `person["1st"]` |

**권장 사용법:**
- 기본적으로 **점 표기법** 사용
- 변수가 필요하거나 특수한 경우에만 **대괄호 표기법** 사용

### 존재하지 않는 속성 접근하기

객체에 없는 속성에 접근하면 **에러가 발생하지 않고** `undefined`가 반환됩니다. 

```javascript
const person = {
  name: "홍길동",
  age: 25
};

console.log(person.city);     // undefined
console.log(person["email"]); // undefined
```

그래서 이런 특성을 활용해 특정한 속성이 객체 안에 있는지 확인할 수도도 있습니다. 

```javascript
const person = {
  name: "홍길동",
  age: 25
};

// 방법 1: undefined 체크
if (person.city === undefined) {
  console.log("city 속성이 없습니다");
}

// 방법 2: in 연산자 (더 명확한 방법)
if ("city" in person) {
  console.log("city 속성이 있습니다");
} else {
  console.log("city 속성이 없습니다");
}
```

### 객체 속성 추가하기

객체를 만든 후에도 언제든지 새로운 속성을 추가할 수 있습니다.

```javascript
const person = {
  name: "홍길동",
  age: 25
};

// 점 표기법으로 추가
person.city = "서울";

// 대괄호 표기법으로 추가
person["hobby"] = "독서";

console.log(person);
// {
//   name: "홍길동",
//   age: 25,
//   city: "서울",
//   hobby: "독서"
// }
```

assing() 메서드를 사용하면 여러 속성을 한번에 추가할 수도 있습니다. 

```javascript
const person = {
  name: "홍길동"
};

// Object.assign() 사용
Object.assign(person, {
  age: 25,
  city: "서울",
  hobby: "독서"
});

console.log(person);
// { name: "홍길동", age: 25, city: "서울", hobby: "독서" }
```

### 객체 속성 수정하기

이미 존재하는 속성의 값을 변경할 수 있습니다. 속성을 추가할 때와 같은 방법을 사용하면 됩니다. 

만일 해당 속성이 없다면 속성이 추가되고 이미 있는 속성이라면 값이 수정됩니다. 

```javascript
const person = {
  name: "홍길동",
  age: 25,
  city: "서울"
};

// 나이 수정
person.age = 26;

// 도시 수정
person["city"] = "부산";

console.log(person);
// { name: "홍길동", age: 26, city: "부산" }
```

---

### 객체 속성 삭제하기

`delete` 키워드를 사용하면 객체의 속성을 삭제할 수 있습니다. 삭제 후에는 속성이 완전히 사라지고 되돌릴 수 없기 때문에 신중하게 사용해야 합니다. 

```javascript
const person = {
  name: "홍길동",
  age: 25,
  city: "서울",
  hobby: "독서"
};

// hobby 속성 삭제
delete person.hobby;

console.log(person.hobby);  // undefined (속성이 없음)

console.log(person); // { name: "홍길동", age: 25, city: "서울" }

// 대괄호 표기법으로도 삭제 가능
delete person["city"];

console.log(person); // { name: "홍길동", age: 25 }
```

### 객체 순회하기 (모든 속성 접근)

객체의 모든 속성을 하나씩 확인하고 싶을 때는 `for...in` 문을 사용합니다.

```javascript
const person = {
  name: "홍길동",
  age: 25,
  city: "서울"
};

for (let key in person) {
  console.log(\`\${key}: \${person[key]}\`);
}

// 출력:
// name: 홍길동
// age: 25
// city: 서울
```

**동작 원리:**
1. `key` 변수에 속성 이름(키)이 하나씩 들어갑니다
2. `person[key]`로 해당 속성의 값에 접근합니다
3. 모든 속성을 순회할 때까지 반복합니다

**실전 예제:**

```javascript
const product = {
  name: "노트북",
  price: 1200000,
  brand: "삼성",
  stock: 5
};

console.log("=== 상품 정보 ===");
for (let key in product) {
  console.log(\`\${key}: \${product[key]}\`);
}

// 출력:
// === 상품 정보 ===
// name: 노트북
// price: 1200000
// brand: 삼성
// stock: 5
```

**활용 팁:** for...in은 객체의 속성 개수를 셀 때도 유용하게 사용할 수 있습니다.

```javascript
const person = {
  name: "홍길동",
  age: 25,
  city: "서울"
};

let count = 0;
for (let key in person) {
  count++;
}

console.log(\`속성 개수: \${count}\`);  // 속성 개수: 3

// 또는 Object.keys()를 사용 (나중에 배울 메서드)
console.log(Object.keys(person).length);  // 3
```

### const로 선언한 객체도 수정 가능? 

`const`로 선언한 객체는 **속성을 수정할 수 있습니다**. 여러 속성으로 이루어진 객체 자체는 수정할 수 없지만 객체 안의 각 속성은 수정할 수 있습니다. 꼭 기억해 두세요.

```javascript
const person = {
  name: "홍길동",
  age: 25
};

// 속성 수정 가능
person.age = 26;
person.city = "서울";

console.log(person);
// { name: "홍길동", age: 26, city: "서울" }

// 객체 자체를 재할당하는 것은 불가능
person = { name: "김철수" };  // 에러!
```

**왜 이런 일이 일어날까?**

`const`는 **변수가 가리키는 메모리 주소**를 변경하지 못하게 합니다. 하지만 **객체 내부의 속성**은 변경할 수 있습니다.

```javascript
// person이라는 변수는 객체가 저장된 메모리 위치를 가리킴
const person = { name: "홍길동" };

// 객체 내부 수정은 OK (메모리 주소는 그대로)
person.name = "김철수";

// 다른 객체로 바꾸려고 하면 에러 (메모리 주소가 바뀜)
person = { name: "이영희" };  // TypeError!
```

**권장:**
- 객체는 대부분 `const`로 선언하세요
- 속성은 자유롭게 추가/수정/삭제할 수 있습니다
- 객체 자체를 다른 것으로 바꿀 일은 거의 없습니다


## 언제 배열을 사용할까?

**순서가 중요할 때**

```javascript
// 요일은 순서가 중요
const days = ["월", "화", "수", "목", "금", "토", "일"];

// 첫 번째 요일
console.log(days[0]);  // "월"

// 마지막 요일
console.log(days[days.length - 1]);  // "일"
```

**같은 종류의 데이터를 여러 개 저장할 때**

```javascript
// 학생 이름 목록
const students = ["홍길동", "김철수", "이영희", "박민수"];

// 점수 목록
const scores = [85, 90, 78, 95, 88];

// 모든 점수의 합계
let sum = 0;
for (let score of scores) {
  sum += score;
}
console.log(sum);  // 436
```

**순서대로 처리해야 할 때**

```javascript
// 할 일 목록 (순서대로 처리)
const todos = ["장보기", "청소하기", "공부하기"];

todos.forEach((todo, index) => {
  console.log(\`\${index + 1}. \${todo}\`);
});
// 1. 장보기
// 2. 청소하기
// 3. 공부하기
```

## 언제 객체를 사용할까?

// 한 사람의 정보
const person = {

```javascript
// 학생의 과목별 점수
const grades = {

```

// 게임 설정
const gameSettings = {

};

if (gameSettings.sound) {
  console.log("소리 ON");
}
