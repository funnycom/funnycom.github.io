## 간단한 프로그램 작성하고 실행하기

가장 간단한 타입스크립트 프로그램부터 시작해봅시다.

### Hello World 작성하기

프로젝트 폴더에 `hello.ts` 파일을 만들고 다음 코드를 작성하세요:

```typescript
// hello.ts
const message: string = "Hello, TypeScript!";
console.log(message);

const name: string = "홍길동";
const age: number = 25;

console.log(\`안녕하세요, 저는 \${age}살 \${name}입니다.\`);
```

**컴파일하고 실행:**

```bash
tsc hello.ts
node hello.js
```

**출력:**
```
Hello, TypeScript!
안녕하세요, 저는 25살 홍길동입니다.
```

### 타입 에러 경험하기

이제 일부러 에러를 만들어봅시다. 타입스크립트가 어떻게 에러를 잡아주는지 확인할 수 있습니다.

```typescript
// error.ts
const name: string = "홍길동";
const age: number = "25";  // 에러!
```

**컴파일 시도:**

```bash
tsc error.ts
```

**에러 메시지:**

```
error TS2322: Type 'string' is not assignable to type 'number'.
```

타입스크립트가 코드를 실행하기 전에 에러를 발견했습니다! 이것이 타입스크립트의 가장 큰 장점입니다.

**에러 수정:**

```typescript
const name: string = "홍길동";
const age: number = 25;  // ✅ 올바른 코드
```

## 타입 표기 연습

### 변수에 타입 지정하기

다양한 타입을 연습해봅시다.

```typescript
// types-practice.ts

// 문자열
const firstName: string = "길동";
const lastName: string = "홍";
const fullName: string = lastName + firstName;
console.log(fullName);  // 홍길동

// 숫자
const age: number = 25;
const height: number = 175.5;
const year: number = 2024;

console.log(\`나이: \${age}세\`);
console.log(\`키: \${height}cm\`);

// 불린
const isStudent: boolean = true;
const isAdult: boolean = age >= 18;

console.log(\`학생입니까? \${isStudent}\`);
console.log(\`성인입니까? \${isAdult}\`);

// 배열
const numbers: number[] = [1, 2, 3, 4, 5];
const fruits: string[] = ["사과", "바나나", "오렌지"];

console.log(\`숫자 배열: \${numbers}\`);
console.log(\`과일 배열: \${fruits}\`);
```

**실행 결과:**

```
홍길동
나이: 25세
키: 175.5cm
학생입니까? true
성인입니까? true
숫자 배열: 1,2,3,4,5
과일 배열: 사과,바나나,오렌지
```

### 타입 추론 vs 타입 표기

타입스크립트는 값을 보고 자동으로 타입을 추론합니다.

```typescript
// 타입 추론 (Type Inference)
let count = 0;           // number로 자동 추론
let message = "Hello";   // string으로 자동 추론
let isOk = true;         // boolean으로 자동 추론

// 타입 표기 (Type Annotation)
let count2: number = 0;
let message2: string = "Hello";
let isOk2: boolean = true;

// 결과는 동일! 타입스크립트가 똑똑하게 추론함
```

**언제 타입을 명시해야 할까?**

```typescript
// ✅ 나중에 값을 할당할 때
let name: string;
name = "홍길동";

// ✅ 빈 배열로 시작할 때
let numbers: number[] = [];
numbers.push(1);
numbers.push(2);

// ✅ 여러 타입이 가능할 때 (나중에 배움)
let data: string | number;
data = "Hello";
data = 123;
```

### 과제 2: 점수 계산하기

세 과목의 점수를 저장하고 총점과 평균을 계산해보세요.

**요구사항:**
- 국어, 수학, 영어 점수를 변수에 저장 (타입 지정)
- 총점 계산
- 평균 계산
- 결과 출력

**힌트:**

```typescript
// scores.ts
const korean: number = // 국어 점수
const math: number = // 수학 점수
const english: number = // 영어 점수

const total: number = // 총점 계산
const average: number = // 평균 계산

console.log(// 결과 출력);
```

#### 정답 2: 점수 계산하기

```typescript
// scores.ts
const korean: number = 85;
const math: number = 90;
const english: number = 88;

const total: number = korean + math + english;
const average: number = total / 3;

console.log("=== 성적표 ===");
console.log(\`국어: \${korean}점\`);
console.log(\`수학: \${math}점\`);
console.log(\`영어: \${english}점\`);
```

**출력:**
```

국어: 85점
수학: 90점
영어: 88점
총점: 263점
평균: 87.67점
```

**설명:**
- `.toFixed(2)`: 소수점 2자리까지 표시

### 과제 3: 쇼핑 목록 만들기

쇼핑 목록을 배열로 만들고 출력해보세요.

**요구사항:**
- 과일 이름을 문자열 배열로 저장
- 과일 가격을 숫자 배열로 저장
- 모든 과일과 가격 출력

**힌트:**

```typescript
// shopping.ts
const fruits: string[] = // 과일 이름들
const prices: number[] = // 과일 가격들

// 출력하기
```

#### 정답 3: 쇼핑 목록 만들기

```typescript
// shopping.ts
const fruits: string[] = ["사과", "바나나", "오렌지", "포도"];
const prices: number[] = [1000, 1500, 2000, 3000];

console.log("=== 쇼핑 목록 ===");

// 방법 1: 인덱스로 접근
console.log(\`\${fruits[0]}: \${prices[0]}원\`);
console.log(\`\${fruits[1]}: \${prices[1]}원\`);
console.log(\`\${fruits[2]}: \${prices[2]}원\`);
console.log(\`\${fruits[3]}: \${prices[3]}원\`);

// 방법 2: for문 사용 (Part 0에서 배운 내용)
console.log("\n=== for문으로 출력 ===");
for (let i = 0; i < fruits.length; i++) {
  console.log(\`\${fruits[i]}: \${prices[i]}원\`);
}

// 총 금액 계산
let total: number = 0;
for (let i = 0; i < prices.length; i++) {
  total = total + prices[i];
}
console.log(\`\n총 금액: \${total}원\`);
```

**출력:**
```
=== 쇼핑 목록 ===
사과: 1000원
바나나: 1500원
오렌지: 2000원
포도: 3000원

=== for문으로 출력 ===
사과: 1000원
바나나: 1500원
오렌지: 2000원
포도: 3000원

총 금액: 7500원
```

### 자가 진단 퀴즈

**1. 다음 중 TypeScript의 장점이 아닌 것은?**

A. 실행 전에 에러를 발견할 수 있다  
B. 자동 완성이 강력하다  
C. 실행 속도가 JavaScript보다 빠르다  
D. 코드의 의도가 명확해진다

**정답: C**

TypeScript는 컴파일 후 JavaScript로 변환되어 실행되므로, 실행 속도는 JavaScript와 동일합니다. TypeScript의 장점은 개발 단계에서 타입 체크를 통해 에러를 미리 발견하고, 개발 생산성을 높이는 것입니다.

**2. 다음 코드 중 올바른 것은?**

\`\`\`typescript
A. const name: string = "홍길동";
B. const age: number = "25";
C. const isOk: boolean = "true";
D. const count: string = 100;
\`\`\`

**정답: A**

- A: ✅ string 타입에 문자열 할당
- B: ❌ number 타입에 문자열 할당
- C: ❌ boolean 타입에 문자열 할당
- D: ❌ string 타입에 숫자 할당

**3. TypeScript 파일의 확장자는?**

A. .js  
B. .ts  
C. .txt  
D. .type

**정답: B**

TypeScript 파일은 `.ts` 확장자를 사용합니다. 컴파일하면 `.js` 파일이 생성됩니다.

**4. 다음 코드의 컴파일 결과는?**

A. 컴파일 에러  
B. 경고만 표시

**정답: C**

타입이 올바르게 지정되어 있으므로 정상적으로 컴파일됩니다.

**5. 다음 중 타입을 명시하지 않아도 되는 경우는?**

```typescript
A. let count = 0;
B. let data;
C. let numbers = [];
D. const name;
```

**정답: A**

`let count = 0;`은 타입스크립트가 자동으로 number로 추론하므로 타입을 명시하지 않아도 됩니다.

- B: 타입을 알 수 없으므로 명시 필요
- C: 빈 배열은 타입 명시 권장
- D: 문법 에러 (const는 선언 시 값 할당 필요)
