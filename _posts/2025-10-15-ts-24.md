## Part 3: 스스로 개선하기 (1-2시간)

이제 AI의 도움을 최소화하고, 스스로 코드를 개선해봅시다.

### 💪 도전 과제

#### Level 1: 기본 개선 (AI 힌트만 사용)

**과제 1-1: 할 일 수정 기능**

```
요구사항:
- updateTodo(id: number, updates: Partial<Todo>): boolean
- title이나 completed를 부분적으로 수정
- 성공하면 true, 실패하면 false

힌트가 필요하면 AI에게:
"updateTodo 메서드의 시그니처와 기본 구조만 알려줘.
구현은 내가 할게."
```

**과제 1-2: 완료된 할 일 일괄 삭제**

```
요구사항:
- clearCompleted(): number
- 완료된 모든 할 일 삭제
- 삭제된 개수 반환

스스로 시도:
1. getFilteredTodos("completed")로 완료된 할 일 찾기
2. 각 할 일의 id로 deleteTodo 호출
3. 삭제 성공한 개수 세기

막히면 AI에게:
"clearCompleted 메서드 구현 방법의 힌트만 줘"
```

**과제 1-3: 우선순위 기능**

```
요구사항:
1. Todo 인터페이스에 priority?: "low" | "medium" | "high" 추가
2. 우선순위별로 필터링하는 메서드
3. 우선순위별로 정렬하는 메서드

스스로 시도해보고, 막히면 단계별로 AI에게 질문:
"Todo 인터페이스에 선택적 속성 추가하는 방법은?"
"priority로 필터링하는 방법은?"
```

---

#### Level 2: API 활용 고급 (부분적 AI 도움)

**과제 2-1: 로딩 상태 관리**

```
요구사항:
1. private isLoading: boolean 속성 추가
2. getLoadingState(): boolean 메서드
3. API 호출 시작 시 true, 종료 시 false
4. 로딩 중에는 중복 요청 방지

스스로 계획:
- 어느 메서드들을 수정해야 하나?
- isLoading을 언제 true/false로 바꿔야 하나?
- finally 블록을 어떻게 활용할까?

부분적 도움:
"API 호출 메서드에서 로딩 상태를 관리하는 패턴을 알려줘"
```

**과제 2-2: 에러 처리 개선**

```
요구사항:
1. ApiError 클래스 만들기
   - statusCode, message 속성
2. ApiService에서 상세한 에러 정보 제공
3. TodoManager에서 에러 타입별 다른 처리

스스로 설계:
- ApiError는 어떤 구조여야 할까?
- 어떤 에러 타입들이 있을까?
- 각 에러를 어떻게 처리할까?

필요시:
"커스텀 Error 클래스 만드는 방법 알려줘"
```

**과제 2-3: 재시도 로직**

```
요구사항:
- API 요청 실패 시 자동으로 재시도
- 최대 3번까지 재시도
- 재시도 간격: 1초, 2초, 3초
- 모두 실패하면 에러

스스로 생각:
- 재시도를 어디서 구현할까?
- 어떻게 딜레이를 줄까?
- 재시도 횟수를 어떻게 셀까?

힌트만:
"Promise를 사용한 재시도 패턴의 기본 구조만 알려줘"
```

#### Level 3: 완전 자율 도전 (AI 도움 최소화)

**과제 3-1: 캐싱 시스템**

```
목표:
- 같은 요청을 반복하지 않도록 캐싱

1. 무엇을 캐싱할까?
2. 캐시를 어떻게 저장할까?
3. 만료 시간을 어떻게 확인할까?
4. 언제 캐시를 사용하고 언제 새로 요청할까?

최후의 질문:
"내가 설계한 캐싱 시스템을 리뷰해줘 [설계 내용]"
```

**과제 3-2: localStorage 동기화**

```
목표:
- 로컬 스토리지에 할 일 저장
- 앱 시작 시 로컬 스토리지에서 로드
- 변경사항 자동 저장
- 서버와 충돌 시 해결 전략

스스로 구현:
1. saveToLocalStorage 메서드
2. loadFromLocalStorage 메서드
3. 자동 저장 로직
4. 서버 데이터와 병합 로직

검증만 요청:
"localStorage 사용법이 맞는지 확인해줘"
```

**과제 3-3: 배치 작업 (Batch Operations)**

```
목표:
- 여러 할 일을 한 번에 서버에 동기화
- Promise.all 활용
- 일부 실패 시 처리 전략

스스로 설계:
- 어떤 메서드들이 필요할까?
- 에러 처리는 어떻게 할까?
- 진행 상태를 어떻게 알릴까?

구현 후 리뷰 요청:
"내 배치 작업 구현을 리뷰해줘 [코드]"
```
