## 원시 타입 vs 참조 타입

자바스크립트의 데이터 타입은 크게 원시 타입과 참조 타입, 두 가지로 나뉩니다:

### 1. 원시 타입 (Primitive Type)
- 숫자, 문자열, 불린, null, undefined
- 변수에 **값 자체**가 저장됩니다

```javascript
const age = 25; // age라는 변수에 25라는 값이 직접 저장됨 

const name = "홍길동";  // name이라는 변수에 "홍길동"이라는 값이 직접 저장됨
```

### 2. 참조 타입 (Reference Type)
- 객체, 배열, 함수
- 변수에 **값이 저장된 메모리 주소**가 저장됩니다. 

다음 코드에서 person이라는 변수에는 객체 자체가 아니라 객체가 저장된 "메모리 주소"가 저장됩니다.

```javascript
const person = { name: "홍길동" };
```

## 메모리 주소란?

컴퓨터의 메모리는 수많은 칸으로 이루어진 거대한 아파트라고 생각하면 됩니다. 각 칸에는 고유한 **주소(번호)**가 있습니다.

```
메모리 구조 예시:

주소(Address)    저장된 값
---------------------------------
0x001           25
0x002           "홍길동"
0x003           { name: "홍길동", age: 25 }  ← 실제 객체
0x004           0x003  ← person 변수 (주소를 가리킴)
```

### 원시 타입의 경우

```javascript
const age = 25;
```

**메모리 상황:**
```
변수 age → [25]
```

`age` 변수는 **25라는 값을 직접** 가지고 있습니다.

```javascript
const age = 25;
age = 30;  // 에러!
```

`const`로 선언했으므로 `age`에 저장된 25를 30으로 바꿀 수 없습니다.

### 참조 타입(객체)의 경우

```javascript
const person = { name: "홍길동", age: 25 };
```

**메모리 상황:**
```
실제 객체:    [메모리 주소 0x1A2B]
             { name: "홍길동", age: 25 }
                      ↑
변수 person → [0x1A2B] (주소를 저장)
```

`person` 변수는 **객체 자체가 아니라, 객체가 있는 위치(주소 0x1A2B)를 가리킵니다**.

### const가 막는 것

`const`는 **변수에 저장된 값(주소)을 바꾸는 것**을 막습니다.

```javascript
const person = { name: "홍길동" };  // person = 0x1A2B

// 가능: 0x1A2B 주소에 있는 객체의 내용 수정
person.name = "김철수";
person.age = 30;

// person은 여전히 0x1A2B를 가리키고 있음 (주소 변경 없음)
// 단지 0x1A2B 위치에 있는 객체의 내용이 바뀐 것
```

**메모리 상황:**
```
실제 객체:    [메모리 주소 0x1A2B]  ← 주소는 그대로
             { name: "김철수", age: 30 }  ← 내용만 변경
                      ↑
변수 person → [0x1A2B]  ← 여전히 같은 주소를 가리킴
```

```javascript
// 불가능: person이 다른 주소를 가리키게 하려는 시도
person = { name: "이영희" };  // 에러!

// 이렇게 하려는 것:
// 새 객체:     [메모리 주소 0x5C7D]
//             { name: "이영희" }
//                     ↑
// person → [0x5C7D]  ← 다른 주소로 바꾸려고 시도 (불가능!)
```

### 그림으로 이해하기

**상황 1: 속성 수정 (가능)**

```javascript
const person = { name: "홍길동" };
person.name = "김철수";
```

```
[Before]
person ---> [주소: 0x1A2B]
            { name: "홍길동" }

[After]
person ---> [주소: 0x1A2B]  ← 주소는 그대로!
            { name: "김철수" }  ← 내용만 변경
```

**상황 2: 객체 재할당 (불가능)**

```javascript
const person = { name: "홍길동" };
person = { name: "김철수" };  // 에러!
```

```
[Before]
person ---> [주소: 0x1A2B]
            { name: "홍길동" }

[시도하는 것]
            [주소: 0x1A2B]
            { name: "홍길동" }

person ---> [주소: 0x5C7D]  ← 다른 주소로 바꾸려고 시도!
            { name: "김철수" }

const는 이것을 막습니다!
```

### 배열도 마찬가지

배열도 객체와 같은 참조 타입입니다.

```javascript
const numbers = [1, 2, 3];

// 배열 내용 수정 가능
numbers[0] = 10;
numbers.push(4);
console.log(numbers);  // [10, 2, 3, 4]

// 배열 자체 재할당 불가
numbers = [5, 6, 7];  // 에러!
```

## 정리

| 타입 | const로 선언 | 값 수정 | 재할당 |
|------|-------------|---------|--------|
| **원시 타입**<br>(숫자, 문자열 등) | `const age = 25` |  불가능 | 불가능 |
| **참조 타입**<br>(객체, 배열) | `const person = {...}` |  가능<br>(내부 속성 수정) |  불가능<br>(다른 객체로 교체) |

**핵심 정리:**

1. **원시 타입**: 변수에 **값 자체**가 저장됨. `const`는 그 값을 바꾸지 못하게 함

2. **참조 타입**: 변수에 **메모리 주소**가 저장됨. `const`는 그 주소를 바꾸지 못하게 함. 하지만 주소가 가리키는 곳의 **내용은 바꿀 수 있음**

```javascript
// 원시 타입
const age = 25;          // age에 25가 저장됨
age = 30;                // 25를 30으로 바꾸려 함 → 불가능

// 참조 타입  
const person = {...};    // person에 주소(예: 0x1A2B)가 저장됨
person.name = "김철수";  //  0x1A2B 위치의 객체 내용 수정 → 가능
person = {...};          // 다른 주소로 바꾸려 함 → 불가능
```
