## 타입스크립트란?

### 자바스크립트의 문제점

자바스크립트는 배우기 쉽고 강력하지만, 몇 가지 문제점이 있습니다.

**문제 1: 실행 전에 타입 에러를 발견할 수 없다**

자바스크립트는 타입을 자동으로 변환하기 때문에 예상치 못한 결과가 나올 수 있습니다. 이런 버그는 코드를 실행해봐야만 발견할 수 있습니다.

```javascript
function add(a, b) {
  return a + b;
}

console.log(add(10, 20));      // 30 
console.log(add("10", "20"));  // "1020" 숫자 더하기를 원했는데...
console.log(add(10, "20"));    // "1020" 예상치 못한 결과
```

**문제 2: 객체의 속성을 잘못 접근해도 모른다**

자바스크립트는 속성 이름을 잘못 쓰거나 없는 속성에 접근해도 에러를 내지 않고 `undefined`를 반환합니다.

```javascript
const user = {
  name: "홍길동",
  age: 25
};

console.log(user.name);   // "홍길동" 
console.log(user.nmae);   // undefined. 오타인데 에러가 안 남
console.log(user.email);  // undefined. 없는 속성인데 에러가 안 남
```

**문제 3: 함수의 매개변수를 잘못 전달해도 모른다**

자바스크립트는 매개변수의 개수나 순서가 틀려도 에러를 내지 않습니다.

```javascript
function greet(name, age) {
  return `안녕하세요, ${age}살 ${name}님!`;
}

console.log(greet("홍길동", 25));        // 정상
console.log(greet(25, "홍길동"));        // 순서가 바뀜
console.log(greet("홍길동"));            // age가 undefined
console.log(greet("홍길동", 25, "서울")); // 인자가 너무 많음
```

**문제 4: 자동 완성이 제한적이다**

```javascript
const product = {
  name: "노트북",
  price: 1000000
};

// 자동 완성을 제안하는 편집기가 product의 속성을 정확히 알 수 없음
product.  // 자동 완성이 나오지 않거나 부정확함
```

### 타입스크립트의 등장

이런 문제들을 해결하기 위해 Microsoft에서 **타입스크립트**를 만들었습니다.

**타입스크립트는 JavaScript에 타입을 더한 언어입니다.**

타입스크립트는 코드를 실행하기 전에 타입을 검사하여 에러를 미리 발견합니다!

```typescript
// TypeScript 코드
function add(a: number, b: number): number {
  return a + b;
}

console.log(add(10, 20));      // 30
console.log(add("10", "20"));  // 컴파일 에러! 실행 전에 발견
console.log(add(10, "20"));    // 컴파일 에러! 실행 전에 발견
```

### 타입스크립트 vs 자바스크립트

| 특징 | 자바스크립트 | 타입스크립트 |
|------|-----------|-----------|
| **타입 시스템** | 없음 (동적 타입) | 있음 (정적 타입) |
| **에러 발견** | 실행 중 (런타임) | 실행 전 (컴파일 타임) |
| **자동 완성** | 제한적 | 강력함 |
| **리팩토링** | 위험함 | 안전함 |
| **학습 곡선** | 쉬움 | 중간 |
| **실행** | 바로 실행 | 컴파일 필요 |

**중요한 사실:**
- TypeScript는 JavaScript의 **상위 집합(Superset)**입니다
- 모든 JavaScript 코드는 유효한 TypeScript 코드입니다
- TypeScript는 최종적으로 JavaScript로 변환되어 실행됩니다

```
JavaScript ⊆ TypeScript

TypeScript 코드 → (컴파일) → JavaScript 코드 → (실행) → 결과
```

## 왜 타입스크립트를 사용할까?

### 1. 타입 안정성 (Type Safety)

**실행 전에 에러를 발견할 수 있습니다.**

버그를 초기에 발견할 수 있고 그만큼 디버깅 시간을 단축합니다. 

```typescript
function divide(a: number, b: number): number {
  return a / b;
}

divide(10, 2);      // ✅ 5
divide(10, "2");    // ❌ 컴파일 에러! 코드 실행 전에 발견
divide(10, 0);      // ⚠️ Infinity (논리적 에러는 직접 처리 필요)
```

### 2. 개발 생산성 향상

**강력한 자동 완성 (IntelliSense)**

에디터가 즉시 에러를 표시하므로 빠르게 수정할 수 있습니다.

```typescript
interface User {
  name: string;
  age: number;
  email: string;
}

const user: User = {
  name: "홍길동",
  age: 25,
  email: "hong@example.com"
};

// user. 을 입력하면 name, age, email이 자동 완성됨!
console.log(user.name);  // 자동 완성으로 빠르게 작성
```

**빠른 에러 피드백**

```typescript
const user: User = {
  name: "홍길동",
  age: 25
  // ❌ email 속성이 없다고 바로 에러 표시!
};
```

### 3. 코드 품질 향상

**코드의 의도가 명확해집니다.**

```javascript
// JavaScript - 의도를 알기 어려움
function processUser(user) {
  // user가 어떤 속성을 가지고 있는지 알 수 없음
}
```

```typescript
// TypeScript - 의도가 명확함
interface User {
  id: number;
  name: string;
  age: number;
}

function processUser(user: User) {
  // user가 id, name, age를 가진다는 것을 명확히 알 수 있음
}
```

**리팩토링이 안전해집니다.**

```typescript
// User 인터페이스를 수정하면
interface User {
  id: number;
  name: string;
  age: number;
  email: string;  // 새 속성 추가
}

// User를 사용하는 모든 곳에서 에러가 표시됨!
// 어디를 수정해야 하는지 바로 알 수 있음
```

### 4. 대규모 프로젝트에 유리

**협업이 쉬워집니다.**

```typescript
// API 응답 타입을 정의하면
interface ApiResponse {
  success: boolean;
  data: User[];
  message: string;
}

// 팀원들이 API 응답 구조를 바로 이해할 수 있음
function fetchUsers(): Promise<ApiResponse> {
  // ...
}
```

**문서화 효과**

타입 자체가 문서 역할을 합니다. 주석 없이도 코드의 의도를 파악할 수 있습니다.

### 5. 최신 JavaScript 기능 사용

TypeScript는 최신 JavaScript 기능을 먼저 지원하고, 이를 구형 브라우저에서도 동작하는 JavaScript로 변환해줍니다.

```typescript
// 최신 문법 사용
const user = {
  name: "홍길동",
  age: 25
};

const { name, age } = user;  // 구조 분해 할당
const newUser = { ...user, city: "서울" };  // 전개 연산자

// → 구형 브라우저에서도 동작하는 JavaScript로 변환됨
```

## 타입스크립트 개발 환경 설정

타입스크립트를 사용하려면 먼저 개발 환경을 설정해야 합니다.

### 1단계: Node.js 설치

타입스크립트를 사용하려면 Node.js가 필요합니다.

**Node.js가 뭔가요?**

지금까지 우리는 웹 브라우저에서 JavaScript를 실행했습니다. 하지만 **Node.js**를 사용하면 컴퓨터에서 직접 JavaScript를 실행할 수 있습니다.

```
브라우저:    웹 페이지에서 JavaScript 실행
Node.js:     컴퓨터(터미널)에서 JavaScript 실행
```

**왜 Node.js가 필요한가요?**

타입스크립트는 JavaScript로 변환(컴파일)하는 과정이 필요합니다. 이 작업을 컴퓨터에서 수행하기 위해 Node.js가 필요합니다.

```
TypeScript 코드 (.ts)
         ↓
    [Node.js 환경]
         ↓
  TypeScript Compiler
         ↓
JavaScript 코드 (.js)
```

**설치 확인:**

터미널(또는 명령 프롬프트)을 열고 다음 명령어를 입력하세요:

```bash
node --version
```

버전이 표시되면 이미 설치되어 있는 것입니다.

```bash
v18.17.0  # 또는 다른 버전
```

**설치가 필요하다면:**

1. [nodejs.org](https://nodejs.org) 방문
2. LTS 버전 다운로드
3. 다운로드한 설치 프로그램 실행
4. 안내에 따라 설치 (기본 설정으로 진행)

**💡 참고:**
- LTS(Long Term Support): 장기 지원 버전, 안정적이고 권장됨
- Current: 최신 버전, 새로운 기능이 있지만 안정성이 떨어질 수 있음

**설치 후 확인:**

터미널을 다시 열고 확인하세요:

```bash
node --version
npm --version
```

두 명령 모두 버전이 표시되면 성공입니다!

### npm이란?

Node.js를 설치하면 **npm**, 즉 **Node Package Manager**도 함께 설치됩니다.

**npm이 하는 일:**

npm은 JavaScript 패키지(라이브러리, 도구)를 설치하고 관리하는 도구입니다. 마치 스마트폰의 앱스토어와 비슷합니다.

```
앱스토어:   앱을 검색하고 설치
npm:        JavaScript 패키지를 검색하고 설치
```

앞으로 공부할 타입스크립트 리액트를 비롯해 각종 개발 도구는 npm을 통해 설치할 것입니다. 

**npm이 없다면** 개발이 매우 불편합니다. 필요한 패키지(라이브러리)가 있을 때마다 해당 웹사이트를 찾아가서 파일을 직접 다운로드해야 합니다. 

또한 패키지들의 버전을 수동으로 기록하고 관리해야 하므로 실수하기 쉽습니다. 

나중에 패키지를 업데이트하려면 다시 웹사이트를 방문해서 새 버전을 찾고, 기존 파일을 지우고, 새 파일을 다운로드하는 복잡한 과정을 거쳐야 합니다.

**npm이 있으면** 모든 것이 간단해집니다. 터미널에서 명령어 한 줄만 입력하면 원하는 패키지가 자동으로 설치됩니다. 

npm은 설치한 패키지의 버전을 자동으로 기록하고 관리해주기 때문에 신경 쓸 필요가 없습니다. 

업데이트가 필요할 때도 간단한 명령어 하나로 모든 패키지를 최신 버전으로 업데이트할 수 있습니다. 마치 스마트폰에서 앱을 설치하고 업데이트하는 것처럼 쉽고 편리합니다.

### 2단계: TypeScript 설치

터미널에서 다음 명령어를 실행하세요:

```bash
npm install -g typescript
```

**설명:**
- `npm`: Node.js 패키지 관리자
- `install`: 설치 명령
- `-g`: 전역(global) 설치 (어디서든 사용 가능)
- `typescript`: 설치할 패키지 이름

**설치 확인:**

```bash
tsc --version
```

```bash
Version 5.3.3  # 또는 다른 버전
```

`tsc`는 TypeScript Compiler의 약자입니다.

### 3단계: 프로젝트 폴더 만들기

```bash
# 새 폴더 만들기
mkdir my-typescript-project

# 폴더로 이동
cd my-typescript-project

# 프로젝트 초기화
npm init -y
```

`npm init -y`는 `package.json` 파일을 자동으로 생성합니다.

### 4단계: 첫 TypeScript 파일 작성

프로젝트 폴더에 `hello.ts` 파일을 만들고 다음 코드를 작성하세요:

```typescript
// hello.ts
function greet(name: string): string {
  return `안녕하세요, ${name}님!`;
}

const message = greet("홍길동");
console.log(message);
```

**중요:**
- TypeScript 파일의 확장자는 `.ts`입니다
- JavaScript는 `.js`, TypeScript는 `.ts`

### 5단계: 컴파일하고 실행하기

**TypeScript를 JavaScript로 컴파일:**

```bash
tsc hello.ts
```

이 명령을 실행하면 `hello.js` 파일이 생성됩니다.

```javascript
// hello.js (생성된 파일)
function greet(name) {
  return "\uC548\uB155\uD558\uC138\uC694, " + name + "\uB2D8!";
}

var message = greet("홍길동");
console.log(message);
```

타입 정보는 사라지고 순수한 JavaScript 코드가 됩니다!

**JavaScript 파일 실행:**

```bash
node hello.js
```

**출력:**
```
안녕하세요, 홍길동님!
```

### 전체 흐름 정리

```
1. hello.ts 작성 (TypeScript 코드)
   ↓
2. tsc hello.ts (컴파일)
   ↓
3. hello.js 생성 (JavaScript 코드)
   ↓
4. node hello.js (실행)
   ↓
5. 결과 출력
```

## 타입스크립트 기본 문법

### 타입 표기 (Type Annotation)

**변수의 타입을 명시적으로 지정하는 것**을 타입 표기라고 합니다.

**기본 문법:**

```typescript
let 변수명: 타입 = 값;
```

**예제:**

```typescript
// 문자열 타입
let name: string = "홍길동";

// 숫자 타입
let age: number = 25;

// 불린 타입
let isStudent: boolean = true;

// 배열 타입
let numbers: number[] = [1, 2, 3, 4, 5];
let fruits: string[] = ["사과", "바나나", "오렌지"];

// 객체 타입
let user: { name: string; age: number } = {
  name: "홍길동",
  age: 25
};
```

**함수의 타입 표기:**

```typescript
// 매개변수와 반환값에 타입 지정
function add(a: number, b: number): number {
  return a + b;
}

// 화살표 함수
const multiply = (a: number, b: number): number => {
  return a * b;
};

// 반환값이 없는 함수 (void)
function sayHello(name: string): void {
  console.log(`안녕하세요, ${name}님!`);
}
```

### 타입 추론 (Type Inference)

TypeScript는 똑똑합니다! 타입을 명시하지 않아도 값을 보고 자동으로 타입을 추론합니다.

```typescript
// 타입을 명시하지 않아도
let name = "홍길동";        // string으로 추론
let age = 25;               // number로 추론
let isStudent = true;       // boolean으로 추론

// 마우스를 올려보면 타입이 표시됨!
```

**함수 반환 타입도 추론:**

```typescript
function add(a: number, b: number) {
  return a + b;  // 반환 타입이 number로 자동 추론됨
}

const result = add(10, 20);  // result의 타입도 number로 추론
```

**언제 타입을 명시해야 할까?**

타입을 언제 명시하고 언제 생략해야 할지 헷갈릴 수 있습니다. 

일반적으로 간단한 변수는 타입스크립트의 타입 추론 기능을 활용하여 타입을 생략해도 괜찮습니다. 예를 들어 `let count = 0`처럼 작성하면 타입스크립트가 자동으로 number 타입으로 인식합니다. 

하지만 함수의 매개변수는 타입스크립트가 추론할 수 없기 때문에 항상 타입을 명시해주어야 합니다. 

그리고 복잡한 구조의 데이터를 다룰 때는 코드를 읽는 사람이 쉽게 이해할 수 있도록 타입을 명시하는 것이 좋습니다.

```typescript
// ✅ 추론이 명확할 때는 생략 가능
let count = 0;
let message = "Hello";

// ✅ 명시하는 것이 좋은 경우
let data: any;  // 나중에 할당할 때
let numbers: number[] = [];  // 빈 배열

// ✅ 함수 매개변수는 항상 명시
function greet(name: string) {  // name의 타입은 추론 불가
  return `Hello, ${name}`;
}

// ✅ 복잡한 타입은 명시
let user: { name: string; age: number; email: string };
```


### 컴파일 과정 이해하기

TypeScript가 JavaScript로 변환되는 과정을 자세히 알아봅시다.

**TypeScript 코드:**

```typescript
// example.ts
function add(a: number, b: number): number {
  return a + b;
}

const result: number = add(10, 20);
console.log(result);
```

**컴파일 명령:**

```bash
tsc example.ts
```

**생성된 JavaScript:**

```javascript
// example.js
function add(a, b) {
  return a + b;
}

var result = add(10, 20);
console.log(result);
```

**변화 포인트:**
1. 타입 표기 (`: number`) 제거
2. `const`가 `var`로 변환 (타겟 버전에 따라 다름)
3. 나머지 로직은 동일

**컴파일 에러 예제:**

```typescript
function add(a: number, b: number): number {
  return a + b;
}

add(10, "20");  // 컴파일 에러!
```

**에러 메시지:**

```
error TS2345: Argument of type 'string' is not assignable to parameter of type 'number'.
```

에러가 있으면 `.js` 파일이 생성되지 않거나, 생성되더라도 에러 메시지가 표시됩니다.
