## 변수 선언의 규칙

### 변수(variable)란
프로그래밍에서 변수란 프로그램을 실행하는 동안 여러번 값이 달라질 수 있는 데이터를 가리킵니다. 반면에 값을 한번 값을 지정하면 값이 바뀌지 않는 데이터를 상수(constant)라고 하죠. 자바스크립트에서는 상수까지 변수에 포함시킵니다.

### 변수 선언의 규칙

**변수를 구별할 수 있도록 이름을 붙여주는 것을 변수 선언이라고 합니다.**

- 변수 이름은 영어 문자와 언더스코어(_ ), 숫자를 사용합니다. 첫글자는 영문자나 언더스코어(_)만 사용할 수 있습니다. (예, now(O) _now(O),  25now(X), $now(X))

- 영어 대소문자를 구별하며 예약어(keyword)는 변수 이름으로 사용할 수 없습니다. 

- 여러 단어를 연결한 변수 이름은 중간에 하이픈(-)사용할 수도 있고 대문자를 섞어 쓸 수도 있는데, 보통 낙타 표기법(camel case)이라고 해서 대문자를 섞어 사용합니다. (예, total-area, totalArea <- 낙타 표기법)

- 프로그래밍할 때는 변수를 수십 개 사용되므로 각 변수의 역할을 일일이 기억하기가 쉽지 않습니다. 변수 이름만 보고도 대충 어떤 값인지 추측할 수 있도록 하는 것이 좋습니다. 


## var를 사용한 변수

ES5까지는 **var** 라는 예약어 다음에 변수 이름을 적어서 변수를 선언했습니다.

```
var currentYear
```

### var 변수의 특징

변수를 선언하고 적용되는 범위를 스코프(scope)라고 하는데, var 변수의 스코프는 함수 블록입니다. 함수 안에서 var를 붙여서 선언하면 해당 함수 안에서만 사용할 수 있습니다. 이렇게 특정 함수 안에서만 사용할 수 있는 변수를 지역 변수 또는 로컬 변수라고 합니다.

```
function addNumber() {
  var sum = 10 + 20;
}
```
그런데 함수 밖에서 변수를 선언하거나 함수 안에서 선언했더라도 변수 이름 앞에 var를 붙이지 않으면 그 변수는 전역 변수(또는 글로벌 변수)가 됩니다. 전역 변수란 프로그램 어디에서든 사용할 수 있는 변수입니다. 문제는 함수 안에서 변수를 선언할 때 실수로 var를 빠뜨릴 수 있다는 것입니다. 원하지 않았는데 전역 변수가 되어 버리는거죠. 예상치 못한 프로그램 결괏값이 나올 수도 있겠죠?
