### 효과적인 학습 방법

**1. 직접 타이핑하기**
- 예제 코드를 복붙하지 말고 직접 타이핑
- 에러를 경험하고 해결하는 과정이 중요

**2. 컴파일 에러 이해하기**
- 타입스크립트 에러 메시지 꼼꼼히 읽기
- 왜 에러가 발생했는지 이해하기

**3. 자바스크립트 코드를 타입스크립트로 변환하기**
- 앞에서 작성했던 코드에 타입 추가해보기
- 점진적으로 타입 적용 범위 넓히기

**4. 공식 문서 활용하기**
- 타입스크립트 공식 핸드북 참고
- 플레이그라운드에서 실험해보기

### AI 활용 팁

**타입스크립트 학습 시 AI에게 물어볼 질문:**

```
- "이 JavaScript 코드를 TypeScript로 변환해줘"
- "이 타입 에러가 왜 발생하는지 설명해줘"
- "interface와 type의 차이를 예제와 함께 알려줘"
- "제네릭을 왜 사용하는지 실전 예제로 보여줘"
```

### 주의사항

**피해야 할 것:**
- any 타입 남용하지 않기
- 타입 에러를 억지로 무시하지 않기 (as any, @ts-ignore)
- 타입 없이 JavaScript처럼 작성하기

**권장사항:**
- strict 모드에서 개발하기
- 명시적 타입 선언 연습하기
- 타입 추론 활용하되, 복잡한 경우 명시하기

## 💡 학습 팁

### TypeScript 시작 단계에서 주의할 점

**1. 에러를 두려워하지 마세요**

에러는 버그를 미리 발견해주는 친구입니다!

```typescript
const age: number = "25";  // ❌ 에러!
// → "아, number 타입에는 숫자만 넣어야 하는구나!"
```

**2. 천천히, 하나씩**

처음부터 복잡한 프로젝트를 만들려고 하지 마세요. 간단한 변수 선언부터 차근차근 연습하세요.

```typescript
// ✅ 먼저 이런 것부터
const name: string = "홍길동";
console.log(name);

// 익숙해지면 점점 확장
const age: number = 25;
const hobbies: string[] = ["독서", "운동"];
```

**3. JavaScript 지식 활용하기**

Part 0에서 배운 JavaScript 지식에 타입만 추가하면 됩니다!

```javascript
// Part 0에서 배운 JavaScript
let count = 0;
count = count + 1;
```

```typescript
// TypeScript로 변환 (타입만 추가)
let count: number = 0;
count = count + 1;
```

**4. 에디터 활용하기**

VS Code 같은 에디터에서는 마우스를 올리면 타입이 표시됩니다. 적극 활용하세요!

### AI 활용 팁

**초보자를 위한 좋은 질문:**

```
1. "다음 JavaScript 코드에 타입을 추가해줘"
   → Part 0에서 작성한 코드 붙여넣기

2. "이 타입 에러가 무슨 의미인가요?"
   → 에러 메시지 복사해서 물어보기

3. "string과 number의 차이가 뭔가요?"
   → 기본 개념 질문

4. "배열에 타입을 지정하는 방법을 알려줘"
   → 구체적인 문법 질문
```

**❌ 피해야 할 질문:**

```
- "타입스크립트로 쇼핑몰 만들어줘"  (너무 복잡)
- "제네릭이 뭐야?"  (아직 배우지 않은 내용)
- "이 코드 고쳐줘"  (구체적인 질문이 아님)
```
